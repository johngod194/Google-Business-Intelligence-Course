# Minnesota Metro Traffic Volume 2012 to 2018

This project summarises a business intelligence visualisation scenario using traffic data from the :contentReference[oaicite:0]{index=0}. The goal is to support infrastructure planning for the Minneapolis interstate by analysing **traffic volume patterns over time**, and how volume changes under **different weather conditions** and **holidays**. :contentReference[oaicite:1]{index=1}

The outcome of this project differs from the capstone-style case studies: instead of formal requirement documents, the focus is on **data preparation + chart design + dashboard organisation + a final presentation deck** communicating findings. :contentReference[oaicite:2]{index=2}

The activities in this exercise follows as per the [Activity Overview:](https://github.com/johngod194/Google-Business-Intelligence-Course/blob/f2420ccca43b32a771e2dabf3083f0e832a47720/Exercices/Minnesota%20Metro%20Traffic%20Volume/Activity%20Overview.pdf)
**Activities:**
1- Collect, clean, and transform the dataset into an analysis-ready table, focusing on key fields such as `date_time`, `traffic_volume`, `weather_main`, and `holiday` (plus any derived fields needed for time-grain analysis). :contentReference[oaicite:3]{index=3}  
2- Build the required visuals in :contentReference[oaicite:4]{index=4} to address stakeholder needs:  
  a) traffic volume across multiple time scales (year → month → week → day → hour),  
  b) volume by weather conditions, and  
  c) volume on holidays (including custom fields/measures if needed). :contentReference[oaicite:5]{index=5}  
3- Organise the charts into a stakeholder-friendly dashboard (supporting comparisons across time and conditions)
